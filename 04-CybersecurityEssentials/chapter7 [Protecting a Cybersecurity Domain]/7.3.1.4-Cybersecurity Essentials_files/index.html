ICMP Abuse
Reconnaissance
Cyber criminals can manipulate ICMP messages to gather the following information:
  • Hosts that are reachable from the Internet.
  • Network topology.
  • The ACL filtering device used by the host.
  • The protocols the host is using.
  • The operating system running on the host.
Denial of Service
Many DoS attacks use ICMP as part of the attack:
  • Sending ICMP messages to a target that are oversized can crash the host since it does not know how to handle these messages.
  • Sending large number of identical fragmented ICMP messages to the target will cause the host to stop responding.
  • Flooding the target with a lot of ICMP messages will degrade the performance of the host and network.
Covert Channel
Many firewalls and networks allow ping traffic to pass through. The use of ping traffic can open up a covert channel which allows the transfer of information objects through existing information channels or networks. It carries the data in small parts making it virtually undetectable by administrators .
RIP Vulnerabilities
  • Advertise bogus routing information.
  • Claim a route to a particular unused host (impersonation).
  • Claim a route to an active host to draw in traffic for inspection or alteration.
  • Compromise address-based authentication protocol.
Defenses Against RIP Attack
  • Filter packets at gateway based on source/destination address.
  • Check the routes before accepting them.
  • Authenticate RIP packet.
  • Review logs.
