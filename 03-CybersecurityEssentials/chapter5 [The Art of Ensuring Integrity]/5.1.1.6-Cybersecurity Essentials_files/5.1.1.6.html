Hashing Passwords

Hashing algorithms turn any amount of data into a fixed-length fingerprint or digital hash. A criminal cannot reverse a digital hash to discover the original input. If the input changes at all, it results in a different hash. This works for protecting passwords. A system needs to store a password in a form that protects it and can still verify that a userâ€™s password is correct.

The figure shows the workflow for user account registration and authentication using a hash-based system. The system never writes the password to the hard drive, it only stores the digital hash.

When a user creates an account, the system hashes the password and stores the hash in a database. When the user later goes to log in with his user and password, the system compares the hash of the password that the user just entered to the hash of the password that was stored in the database. If the hashes match, the system grants the user access.
